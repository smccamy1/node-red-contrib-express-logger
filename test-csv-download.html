Implementation Details

The solution implements a dual-endpoint approach:

  1. Authenticated Endpoint: /express-logger/:id/download-csv/:fileName - Uses Node-RED's httpAdmin with authentication
  2. Public Endpoint: /express-logger-download/:id/:fileName - Uses Node-RED's httpNode without authentication
  3. Fallback Logic: The frontend tries the authenticated endpoint first, then falls back to the public endpoint if authentication fails

Security Considerations

  • Public endpoint only serves files from the designated CSV export directory
  • Path traversal protection prevents access to files outside the export directory
  • Files are only accessible if they exist in the configured export path
  • Consider adding token-based access or time-limited URLs for additional security
